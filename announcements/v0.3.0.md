# Tidepool v0.3.0 — Helius Historical APIs catch-up

**Two new JSON-RPC methods. Same wire shapes as Helius shipped this spring.**

Since Tidepool v0.1.x, Helius has shipped a stream of new APIs out of their Historical APIs team: `getTransactionsForAddress` in early 2026, `getTransfersByAddress` shortly after, with updates landing as recently as last week (full Solana history from genesis, p-token batch transfer parsing). Tidepool's "Helius-compatible" claim was starting to age. v0.3.0 closes the gap on the two headline methods.

## What's new

### `getTransfersByAddress`

Parsed token + SOL transfer history for a wallet, returned as a flat list of normalized events. No more looping `getSignaturesForAddress` + `getTransaction` + parsing pre/post balances yourself.

```jsonc
{
  "method": "getTransfersByAddress",
  "params": [
    "<WALLET>",
    {
      "mint": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",  // optional
      "direction": "in",                                       // "in" | "out" | omitted
      "limit": 100,
      "sort": "asc",                                           // "asc" | "desc"
      "paginationToken": "<cursor>"                            // optional
    }
  ]
}
```

Response shape matches Helius exactly:

```jsonc
{
  "result": {
    "data": [
      {
        "signature": "...",
        "slot": 12483921,
        "blockTime": 1616745600,
        "type": "transfer",
        "fromUserAccount": "...",
        "toUserAccount": "<WALLET>",
        "fromTokenAccount": "...",
        "toTokenAccount": "...",
        "mint": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
        "amount": "1000000",
        "decimals": 6,
        "uiAmount": "1",
        "confirmationStatus": "finalized"
      }
    ],
    "paginationToken": "..."
  }
}
```

### `getTransactionsForAddress`

Combined sig fetch + tx fetch + classify in one call. Returns full enhanced transactions (not just signatures). Supersedes the older REST `getTransactionsByAddress`.

```jsonc
{
  "method": "getTransactionsForAddress",
  "params": [
    "<ADDRESS>",
    {
      "limit": 100,
      "paginationToken": "<cursor>",
      "minSlot": 1000000,
      "maxSlot": 2000000,
      "status": "success"   // "success" | "failure" | omitted
    }
  ]
}
```

Both methods are compat-classed **BEST_EFFORT** — the response *shape* matches Helius exactly, but the *coverage* is limited to whatever transactions Surfpool has streamed. Helius's claim of "full Solana history from genesis" requires their archival backend; Tidepool can't reproduce that locally and doesn't try. The manifest entries document this clearly so consumers know what they're getting.

## What this isn't (yet)

- **No Wallet REST API** (`/v1/wallet/{wallet}/balances`, `/transfers`, `/history`, `/funded-by`, `/identity`). New transport surface, separate work, v0.4 candidate.
- **No `getProgramAccountsV2` / `getTokenAccountsByOwnerV2`** — wait, scratch that, those have been in since v0.2. The research doc was wrong.
- **No MCP variant.** That's the standalone "Helius for Agents" mirror — separate sub-arc.
- **No LaserStream-equivalent event emitter.** Streaming as the primary product is where Helius is heading; Tidepool's webhook-CRUD simulator is still a generation behind there. Larger work, future release.

## Install

```bash
cargo install tidepool-cli         # binary
npm install @vibestackmd/tidepool  # JS test integrations (MSW, Nock, undici, anything)
```

Lockstep across both registries at 0.3.0. Provenance attestations on the npm tarball, OIDC-published from GitHub Actions.

## Try it

If you have a real Helius API key and want to spot-check shape compatibility, hit `https://transfers-demo.helius.dev` for a known-good response, then compare to Tidepool serving the same wallet (limited by what your local validator has). Same JSON, same field names, same casing.

## What's next

v0.4 is the **Wallet REST** + **MCP variant** arc. Wallet REST gives us our first `/v1/` transport route tree; the MCP variant ships `tidepoolWallet` / `tidepoolAsset` / `tidepoolTransaction` tools that drop directly into Helius's Claude Code plugin shape.
